# Инструкция по ведению Wiki-LLM (по модели Андрея Карпатого)

Этот файл содержит правила, соглашения и описания операций для поддержания структуры и актуальности нашей базы знаний о промышленной безопасности и охране труда.

---

## Архитектура базы знаний

База знаний состоит из трех уровней:

1. **raw/** — Неизменяемые исходные текстовые файлы статей, сгруппированные по тематическим категориям. Это единственный источник правды.
2. **wiki/** — Динамическая база знаний из взаимосвязанных markdown-файлов:
   - `wiki/index.md` — Оглавление и каталог сущностей (терминов, категорий, ключевых тем).
   - `wiki/log.md` — append-only лог изменений (импорты, перекрестные ссылки, результаты линтинга).
   - `wiki/<Сущность>.md` — Страницы терминов, законов и понятий (например, `СОУТ.md`, `ЕТН.md`), содержащие обобщенную информацию и ссылки на исходные статьи.
3. **CLAUDE.md** — Эта инструкция.

---

## Операции над базой знаний

### 1. Ingest (Импорт / Добавление источника)
Когда пользователь просит добавить статью из `raw/` или новую информацию:
1. Прочтите исходный файл.
2. Выделите ключевые сущности, упомянутые в статье (например, законы, штрафы, процедуры, термины).
3. Обновите существующие страницы сущностей в `wiki/` (или создайте новые, если сущности еще нет):
   - Добавьте краткую выжимку новой информации.
   - Добавьте ссылку на исходную статью формата `[Название статьи](file:///absolute/path/to/wiki-llm/raw/Категория/Файл.txt)`.
4. Обновите `wiki/index.md`, если была добавлена новая сущность.
5. Запишите операцию в `wiki/log.md` в формате:
   ```markdown
   ## [ГГГГ-ММ-ДД] ingest | <Название статьи>
   - Обновлены сущности: [[Сущность1]], [[Сущность2]]
   - Добавлены новые сущности: [[Сущность3]]
   ```

### 2. Lint (Проверка целостности)
Периодически запускайте проверку базы знаний:
- Наличие "сиротских" страниц (на которые нет ссылок из других мест, включая `index.md`).
- Неработающие ссылки на файлы из `raw/` или другие вики-страницы.
- Противоречия между старыми и новыми версиями нормативно-правовых актов (НПА), описанные в разных статьях (выносите противоречия или обновления законов на страницы соответствующих сущностей с пометкой `[!IMPORTANT]`).

### 3. Query (Запрос информации)
При ответе на вопросы пользователя:
1. Сначала просмотрите `wiki/index.md` для поиска подходящих концепций и сущностей.
2. Прочитайте соответствующие страницы в `wiki/`.
3. Сформулируйте ответ со ссылками на исходные файлы из `raw/` и страницы сущностей из `wiki/`.

## Шаблон страницы сущности (`wiki/<Название>.md`)

```markdown
# Название сущности / Концепции

Краткое понятное определение концепции или термина.

## Ключевая информация

Детальное описание (правила проведения, нормативные требования, штрафы и т.д.).

## Связанные статьи и источники

- [<Название статьи 1>](file:///absolute/path/to/wiki-llm/raw/Категория/Файл1.txt) — Краткое описание того, о чем говорится в статье применительно к данной теме.
- [<Название статьи 2>](file:///absolute/path/to/wiki-llm/raw/Категория/Файл2.txt) — ...

## Связанные темы
- [[Другая Сущность 1]]
- [[Другая Сущность 2]]
```
